API Documentation

1. "rest/stories/" -->>

@GET:
gets the function to return a list of all the DBObjects i.e stories created so far
@POST:
Creates a New story object and saves it in the database
Parameters consumed : { title : "" , storytext : "" }
where both the parameters are set by the user.

2. "rest/stories/{title}" -->>

@GET :
this rest service is used to get the single story information in order to continue that story.


3. "rest/stories/search/{str}"-->>

@GET :
Search rest service created which can search by keyword in title field only..!!


4. "rest/stories/addstory" -->>

@POST :
this rest servie is used to update the existing story object in the database when the user adds the new storyline to the story.
Parameters consumed : { title : "" , storytext : "" }
Here the title is already set by getting the information from the rest service "rest/stories/{title}"
And the title is set by user to add on to the already existing story
